# Model Coexistence — 鲌类同域共存与生态位分化

> **触发**: 用户查询鲌类同域共存、生态位分化、资源分割、种间竞争
> **方法**: 生态位量化 + 资源分割分析 + 共存机制检验

## PREFLIGHT

1. 确认共存物种对: C. alburnus ↔ C. mongolicus / C. oxycephalus / Ch. erythropterus
2. 收集数据: 出现位点、环境变量、同位素、胃含物
3. 确定分析维度: 空间 / 食性 / 时间

## 同域共存物种对

### 对 1: 翘嘴鲌 ↔ 蒙古鲌 (最常见)

```
分布重叠水体: 洞庭湖、鄱阳湖、太湖、梁子湖

共存机制 (三维生态位分化):

空间维度:
  - 翘嘴鲌: 开阔水域中上层 (0-3m)，距岸 > 100m
  - 蒙古鲌: 近岸带中下层 (2-5m)，距岸 20-80m
  - 水平重叠: 中等 (~40% 分布区重叠)

食性维度:
  - 翘嘴鲌: 偏好较大猎物 (> 8 cm 鱼类)，Pianka 重叠 0.5-0.7
  - 蒙古鲌: 偏好较小猎物 (< 6 cm 鱼类 + 虾类)
  - δ¹³C 差异: 蒙古鲌偏正 (~1-2‰)，反映更多沿岸带碳源

时间维度:
  - 翘嘴鲌: 晨昏活动高峰 (06:00-08:00, 17:00-19:00)
  - 蒙古鲌: 昼夜均有摄食活动
  - 繁殖期: 翘嘴鲌 5-7月 vs 蒙古鲌 5-6月 (部分重叠)
```

### 对 2: 翘嘴鲌 ↔ 尖头鲌

```
分布重叠: 长江中上游干支流

共存机制:
  - 微生境: 翘嘴鲌↗主流，尖头鲌↗回流区
  - 体型分化: 尖头鲌体型较小 (max 50 cm)，猎物谱不同
  - 研究最薄弱: 尖头鲌基础生态学数据严重不足
```

### 对 3: 翘嘴鲌 ↔ 红鳍原鲌

```
分布重叠: 长江中下游 + 珠江

共存机制:
  - 红鳍原鲌食性更广 (杂食偏动物性): 营养级低 0.3-0.5
  - 低氧耐受: 红鳍原鲌 > 翘嘴鲌
  - 空间分化: 红鳍原鲌偏好浅水草区
```

## 生态位量化方法

```
生态位宽度:
  - Levin's B: B = 1 / Σ(p_i²)
  - 标准化: Ba = (B - 1) / (n - 1)
  - 0 = 特化, 1 = 泛化

生态位重叠:
  - Pianka (1973): O_jk → 见 analyze-trophic
  - 零模型检验: 随机化 (1000 次) → 观测值 vs 期望值
  - E-statistic: 生态位等价性检验

多维生态位:
  - 多维超体积 (n-dimensional hypervolume)
  - R 包: hypervolume
  - PCA 前 2-3 个主成分 + 核密度估计
  - 重叠体积 / 联合体积
```

## 资源分割分析

```
饵料大小分布:
  - Kernel density 估计: 猎物体长分布
  - Kolmogorov-Smirnov 检验: 分布差异显著性
  - 预期: 翘嘴鲌猎物偏大 (右移)

觅食生境:
  - δ¹³C 双源混合模型: 沿岸带 vs 湖心带饵料比例
  - 预期: 蒙古鲌沿岸带比例 > 翘嘴鲌

时间分割:
  - 方法: 24h 连续采样 / 水下声学遥测
  - Diel 活动指数: 昼/夜 捕获量比
  - 预期: 翘嘴鲌更偏晨昏型
```

## 共存机制分类

```
经典机制:
  1. 资源分割 (Resource partitioning)
     — 猎物大小/类型/空间来源差异

  2. 栖位分化 (Habitat segregation)
     — 空间分布/微生境选择差异

  3. 时间分割 (Temporal segregation)
     — 活动节律/繁殖物候差异

  4. 条件依赖性共存 (Condition-dependent)
     — 环境波动 → 竞争排斥 ↔ 共存交替

检验框架:
  H0: 物种间生态位无差异
  H1: 至少一个维度存在显著分化

  IF Pianka overlap < 0.6 OR null model p < 0.05:
    → 支持资源分割假说
```

## 环境变化对共存的影响

```
水位调控:
  - 水位下降 → 沿岸带收缩 → 蒙古鲌栖息地减少
  - 空间重叠增加 → 竞争加剧 → 蒙古鲌可能被排斥

富营养化:
  - 透明度降低 → 视觉捕食者 (翘嘴鲌) 摄食效率下降
  - 低氧区扩大 → 红鳍原鲌相对受益

禁捕:
  - 所有鲌类种群恢复 → 种间竞争强度增加?
  - 需要长期监测
```
